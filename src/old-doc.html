概述

DTHybirdEngine1.0是云宝宝大数据开发的一套跨平台的混合开发框架，目前框架支持iOS和Android两个移动端平台。本框架提供丰富的移动端硬件API（震动、摄像头、麦克风、GPS、硬件唯一标示、WIFI、电话、短信等）、原生控件API（包括弹框、输入框、进度条、下拉刷新、地图）、启动器API、认证、城市等能力。并整合开发过程中涉及到的相关资源及预览工具，以DemoApp的方式供开发者下载使用。

引入 jssdk.js 会得到一个全局变量dd，支持amd、cmd引入方式 注意：js文件版本在添加升级功能时地址会变化，如有需要（比如要使用新增的js-api）,请随时关注地址变更。但是旧版本js文件也将一直可用。

接口约定

  1. 所有接口都为异步
  2. 接受一个 object 类型的参数
  3. 成功回调 onSuccess (某些异步接口的成功回调，将在事件触发时被调用，具体详情请查看相关 onSuccess 回调时机，未做描述的即为同步接口)
  4. 失败回调 onFail
dd.命名空间.功能.方法({
    参数1: '',
    参数2: '',
    onSuccess: function(data) {
    //成功回调 
    },
    onFail: function(error){
    //失败回调 
    }
})

返回说明

参数 说明
data JSON 字符串
error 一般情况下为 String, 具体的视接口而定

音频

开始录音

启动语音录制，支持最长为60秒（包括）的音频录制。

dd.device.audio.startRecord({
    onSuccess : function () {},
    onFail : function (error) {}
});

停止录音

停止语音录制，同时将录制的语音上传到服务端，返回音频资源的MediaID。返回音频的MediaID，可用于本地播放和音频下载

dd.device.audio.stopRecord({
    onSuccess : function(data){
        // data => { 
        //   mediaId: 1500980530298.aac 
        //   duration: 3 
        // } 
    },
    onFail : function (error) {}
});

返回说明

参数 说明
data.mediaId 返回音频的MediaID，可用于本地播放和音频下载
data.duration 返回音频的时长，单位：秒

监听录音自动停止

当语音录制时间超过60秒时，App会自动停止语音录制，同时将录制的语音上传到服务端，返回音频资源的MediaID。 推荐在调用 dd.device.audio.startRecord 前设置监听录音自动停止的回调。

dd.device.audio.onRecordEnd({
    onSuccess: function(data) {
        // data => { 
        //   mediaId: 1500980530298.aac 
        //   duration: 3 
        // } 
    },
    onFail: function (error) {}
});

返回说明

参数 说明
data.mediaId 停止播放音频的 MediaID
data.duration 返回音频的时长，单位：秒

参数说明

参数 参数类型 说明

返回说明

所要播放的音频的 localAudioId 的值可以通过两种途径获得:

  1. 通过 dd.device.audio.stopRecord 或 dd.device.audio.onRecordEnd 获取录制的音频的 MediaId
  2. 通过 dd.device.audio.download 下载服务端音频资源获取 localAudioId
dd.device.audio.play({
    localAudioId: "1500980530298.aac",
    onSuccess: function () {},
    onFail: function (error) {}
});

支付

支付宝支付

实际的支付结果以后端查询接口为准,这里成功回调只能说明支付宝这边支付调用成功,相关的数据不一定同步到后端. 如果是失败回调,可以通过返回的数字,可以获得一些反馈

dd.biz.util.pay({
    platform:'1' ,
    arg:{
        "credential":"等待支付的订单信息"
      },
    onSuccess: function(data) {},
    onFail: function(error) {
        // error = "8000" 
    }
})

参数说明

参数 参数类型 说明
platform String 支付平台，“1”为支付宝，“2”为微信支付（暂不支持）
arg.credential String 等待支付的订单信息

返回说明

参数 说明
error 调用支付宝支付过程中发生错误的错误码.
error="8000" 正在处理中，支付结果未知（有可能已经支付成功），请查询商户订单列表中订单的支付状态
error="4000" 订单支付失败
error="5000" 重复请求
error="6001" 用户中途取消
error="6002" 网络连接出错
error="6004" 支付结果未知（有可能已经支付成功），请查询商户订单列表中订单的支付状态
